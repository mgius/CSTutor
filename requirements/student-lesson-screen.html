2.4. Using the Student Lesson Screen

This scenario introduces the interactive code lesson interface. Interactive coding lessons are lessons that allow users the opportunity to modify and execute provided code and see output immediately.

2.4.1 Selecting a Lesson

After logging in or choosing anonymous access (see Logging In), the student will be presented with the main lesson screen. The leftmost frame presents each lesson and sublesson in a tree structure. The student can select a lesson or or sublesson from the navigation pane. Selecting a lesson will populate the interactive code shell and the lesson pane.
After selecting a lesson, a student may select a new lesson in the navigation pane at any time.



Figure 1: Initial Lesson Screen

2.4.2 Interactive Code Lesson

The lesson pane and the interactive code shell are now populated, and the student can now read the text and images presented to them. The interactive code shell (discussed in more detail below) can be used to verify understanding of concepts and examples in the lesson.

2.4.3 Interactive Code Shell

Figure 2: Interactive Java Code Shell


Lessons can include an interactive code shell to allow the student to experiment with the content of the lesson. The code shell can optionally be populated by the lesson creator. The code shell (examples show a Java shell) take lines of code one at a time, and execute them upon entering.
The shell is not restricted to code specified by examples, giving the student freedom to experiment within the confines of the code shell language.
While not applicable to the Java shell, an "execute" button may be present for languages where an interactive shell is not available.
