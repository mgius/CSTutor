4. Developer Overview

When implementing CSTutor, there are a few things to note before beginning

  • Class/Lesson/Quiz Hierarchy
  • Executable Code
  • Published vs. Saved Pages

4.1 Class/Lesson/Quiz Hierarchy

The Hierarchy of pages, or "Page Tree" is the overall structure of Classes.
  • As noted in the specification, a Class is always the root of a Page Tree and the root of a Page Tree is always a Class. That is, a Class never is a subpage and no Lesson or Quiz is a root of a Page Tree.
  • Within the Page Tree each node can have any amount of Lessons and Quizzes. As noted in the specification, Lessons may have lessons and quizzes within them. However, Quizzes never have sublessons or subquizzes. That is, a quiz always is a leaf on the Page Tree.
  • As noted in the specification, each Page contains two other Page IDs that represent the previous and next Pages. This next/previous structure follows a Preorder traversal of the Page Tree. Pushing "Next Page" takes the user to the next node in the traversal and "Prev Page" takes the user to the previous node in the traversal.

4.2 Executable Code

Executable code is a difficult section to implement, especially if CSTutor is web application.
  • There is a potential for security issues when unknown people are writing and executing code.
  • There is a potential for performance issues with non-terminating programs.
  • Providing a reasonable response time is expected.

4.3 Published vs. Saved Pages

It is important to distinguish between the Published version of a Page and the currently Saved version.
  • The Published version of a Page is visible to all who are enrolled in the Class.
  • The Saved version of the Page is the version that one of the owners is editing.
  • This opens up versioning issues with multiple editors. For simplicity's sake, we recommend using a "Last Published Wins" philosophy. That is, when a user publishes a Saved page, it replaces the current Published page.
