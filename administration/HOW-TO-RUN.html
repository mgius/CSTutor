Hosted website environment

Our project is hosted at https://autotutor.csc.calpoly.edu/demo/

Test data has been loaded. Several users exist that can be logged in. Of note are the "mgius" user, who is the course owner of the CPE102 course, and "jinloes" who is the course owner of the CPE103 course.
Creating new users/courses works, and creating new lessons is working as well.
Debugging output has been left enabled, so "error" pages will include ugly python stack traces.

Known Bugs

Our ticket tracker is available at https://autotutor.csc.calpoly.edu/trac/report/1 if you would like to view all open issues. Notable issues for this release include:

  • The "edit" button does not appear for all locations.
  • Most statistic related functions do not work.
  • The course/page list on the left hand side is ugly and out of order.
  • Editing Quizzes isn't really implemented yet (although the interface is laid out)
  • Permissions are only half implemented. Many operations that should be priviledged are currently not.
  • Anonymous usage is not yet working.
  • The site is generally ugly.

How To Run Development Website

  1. Install Django
    • Download the tar from http://www.djangoproject.com/download/
    • Run these commands where you download it:
    • tar xzvf Django-1.1.1.tar.gz
    • cd Django-1.1.1
    • sudo python setup.py install
      • NOTE: These instructions are also in the link above
  2. Install Django Debug Toolbar
    • Run this command: easy_install django-debug-toolbar
  3. Setup up database and Web server
    • Within the implementation folder, run these commands:
      • make clean
      • make database
      • make runserver
    • If you ever need to rebuild the database don't forget to run make cleandatabase before building the database again
    • Once it is running, you should see a line in the output that is similar to below
    • Development server is running at http://127.0.0.1:8000/
    • You should be able to access our website at that url.
